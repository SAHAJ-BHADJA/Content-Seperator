Project: Multimodal Segmentation of Long-Form Online Video into Content and Non-Content

1. Project Overview

This project implements an end-to-end system that automatically segments long-form videos into core content and non-content (ads, intros, outros, etc.) using multimodal analysis of visual, audio, and speech signals.

Key Features

  • Multimodal Analysis: Combines visual features, audio features, and speech transcripts
  • Interactive Video Player: Custom player with segment timeline and navigation
  • Auto-Skip Functionality: Automatically skip detected advertisements
  • Export Capabilities: YouTube chapters, HTML reports, CSV, SRT subtitles
  • Speech Analysis: Whisper-based transcript analysis for ad keyword detection

4. Multimodal Signal Analysis

4.1 Visual Analysis (video_analyzer.py)

Feature Description Purpose
Color Histogram HSV color distribution (Hue, Saturation, Value) Detect visual style changes (ads often have different color schemes)
Scene Change Detection Histogram correlation between consecutive frames Identify segment boundaries (cuts between content and ads)
Motion Estimation Frame differencing / Optical flow magnitude Detect static screens (intros, holding screens, transitions)
Brightness/Contrast Mean luminance and standard deviation Identify black screens, fade transitions
Edge Density Canny edge detection ratio Distinguish text overlays from natural content
Dominant Colors K-means clustering on pixel colors Detect branded content (sponsor logos have distinct colors)

4.2 Audio Analysis (audio_analyzer.py)

Feature Description Purpose
RMS Energy Root Mean Square of audio amplitude Detect silence, low-energy segments, dead air
Spectral Centroid Center of mass of the frequency spectrum Distinguish music (higher) from speech (lower)
Zero Crossing Rate Rate of sign changes in the signal Differentiate speech from music/noise
Spectral Rolloff Frequency below which 85% of energy exists Detect music segments (intros often have music)
Spectral Flatness How noise-like vs tonal the audio is Identify background music vs speech
Music Probability Combined metric estimating music presence Flag intro/outro segments with background music

4.3 Speech Analysis (transcript_analyzer.py)

Feature Description Purpose
Speech-to-Text OpenAI Whisper transcription Convert spoken words to searchable text
Ad Keyword Detection Match against known ad phrases "Sponsored by", "use code", "check the link"
Intro Keywords Detect greeting patterns "Welcome back", "in this video", "let's get started"
Outro Keywords Detect closing patterns "Thanks for watching", "see you next time", "subscribe"

5. How Signals Combine for Classification

Multimodal Fusion Strategy: The system uses a rule-based classifier that combines evidence from all three modalities. Each modality provides confidence scores, and the final classification is determined by weighted voting.

Classification Logic

def classify_segment(visual_features, audio_features, speech_features):
    
    # 1. Check for silence/dead air (Audio dominant)
    if audio.rms_energy < SILENCE_THRESHOLD:
        return "SILENCE"
    
    # 2. Check for intro/outro (Music + Keywords)
    if audio.music_probability > 0.7 and is_near_start:
        return "INTRO"
    if audio.music_probability > 0.7 and is_near_end:
        return "OUTRO"
    
    # 3. Check for ads (Speech keywords + Visual changes)
    if speech.has_ad_keywords or visual.scene_change_detected:
        if speech.confidence > 0.8:
            return "AD"
    
    # 4. Check for transitions (Low motion + Scene change)
    if visual.motion < LOW_MOTION_THRESHOLD and visual.is_static:
        return "TRANSITION"
    
    # 5. Default to core content
    return "CORE_CONTENT"

Feature Weighting

Segment Type Primary Signal Supporting Signals
Advertisement Speech keywords (60%) Scene changes (25%), Color shift (15%)
Intro Music detection (50%) Position in video (30%), Speech keywords (20%)
Outro Music detection (50%) Position in video (30%), Speech keywords (20%)
Silence Audio RMS energy (90%) Visual motion (10%)
Transition Visual motion (60%) Scene change (40%)

7. Expected Questions & Answers

Q1: How does your system detect advertisements?

Answer: We use a multimodal approach combining three signals:

  • Speech: Whisper transcribes audio, then we search for ad keywords like "sponsored by", "use code", "check the link below"
  • Visual: Scene change detection identifies cuts between main content and ad segments
  • Audio: Spectral analysis detects changes in audio characteristics (ads often have different audio mixing)

Q2: Why use multiple modalities instead of just one?

Answer: No single modality is sufficient:

  • Visual alone fails when ads look similar to content
  • Audio alone fails when there's no music transition
  • Speech alone fails for visual-only ads or non-English content
  • Combining them provides robustness through redundancy

Q3: How do you detect scene changes?

Answer: We compute color histograms (HSV) for consecutive frames and calculate their correlation. When correlation drops below a threshold (e.g., 0.7), we mark it as a scene change. Large drops indicate hard cuts, smaller drops indicate gradual transitions.

Q4: What is the Spectral Centroid and why is it useful?

Answer: Spectral centroid is the "center of mass" of the frequency spectrum - essentially where most audio energy is concentrated. Music typically has a higher centroid (more high frequencies) than speech. We use this to detect intro/outro music segments.

Q5: How does motion estimation work?

Answer: We use two methods:

  • Fast mode: Frame differencing - subtract consecutive grayscale frames and measure average pixel difference
  • Accurate mode: Optical flow (Farneback algorithm) - computes pixel motion vectors between frames

Low motion indicates static screens like intros, transitions, or holding screens.

Q6: How do you handle synchronization between video and audio?

Answer: We use OpenCV for video frames and Pygame for audio playback. When seeking, we update both simultaneously. The video thread emits the current position, and we sync the audio player to match. Pygame's mixer allows precise seeking in milliseconds.

Q7: What is Zero Crossing Rate and what does it indicate?

Answer: ZCR measures how often the audio signal crosses zero amplitude per second. Speech has moderate ZCR (periodic vocal patterns), music has variable ZCR, and noise has high ZCR. This helps distinguish speech from background music.

Q8: How accurate is your system?

Answer: Accuracy depends on the video type. For videos with clear ad transitions and spoken sponsorship mentions, we achieve high precision. The system includes an evaluation module that computes Precision, Recall, F1-score, and IoU against ground truth annotations.

Q9: Why did you use OpenCV instead of Qt's built-in media player?

Answer: Qt's QMediaPlayer relies on system codecs which vary across platforms. On Windows, it often fails with certain video formats. OpenCV provides consistent cross-platform video decoding using its own backends (FFmpeg), ensuring reliable playback regardless of system configuration.

Q10: How would you improve the system if you had more time?

Answer: Potential improvements include:

  • Deep learning models (CNN for visual, LSTM for temporal patterns)
  • Cross-episode repetition detection for recurring intros
  • User feedback loop to improve classification over time
  • Real-time processing for live streams
  • OCR to detect sponsor text overlays

Q11: What is the Music Probability metric?

Answer: It's a combined score from multiple audio features:

music_prob = 0.4 * normalized_spectral_centroid 
           + 0.3 * spectral_rolloff_ratio 
           + 0.3 * (1 - spectral_flatness)

High centroid, high rolloff, and low flatness together indicate music presence.

Q12: How do you handle videos in languages other than English?

Answer: Whisper supports 99+ languages and auto-detects the language. The keyword detection currently focuses on English phrases, but the visual and audio analysis is language-independent. The system can be extended with multilingual keyword lists.

Q13: What export formats does your system support?

Answer:

  • YouTube Chapters: Timestamp format for video descriptions
  • HTML Report: Visual timeline with statistics
  • CSV: Spreadsheet-compatible segment data
  • SRT: Subtitle format with chapter markers
  • JSON: Full metadata for programmatic use
  • EDL: Edit Decision List for video editors

Q14: How does the auto-skip feature work?

Answer: During playback, we continuously check if the current position falls within an "AD" segment. If auto-skip is enabled and we enter an ad segment, we immediately seek to the segment's end time and continue playing. The status bar shows which ad was skipped.

Q15: What is the timeline showing in the player?

Answer: The timeline is a visual representation of the entire video with color-coded segments:

  • Green: Core content (main video)
  • Red: Advertisement/Sponsorship
  • Blue: Intro
  • Purple: Outro
  • Yellow: Transition
  • Gray: Silence/Dead air

Clicking anywhere on the timeline seeks to that position.

9. Conclusion

This project demonstrates a practical multimodal approach to video segmentation. By combining visual analysis (color, motion, scene changes), audio analysis (energy, spectral features), and speech analysis (keyword detection), the system can effectively identify and classify non-content segments in long-form videos.

The interactive player provides a user-friendly interface for navigating segmented videos, while the export features enable integration with platforms like YouTube and video editing software.

Key Takeaway: Multimodal fusion is essential because no single signal reliably identifies all types of non-content. The combination of visual, audio, and speech analysis provides robust detection across diverse video types.
